# Aerial Object Classification & Detection

Project contents:
- `Aerial Object Classification.ipynb` — end-to-end notebook: dataset inspection, preprocessing, baseline Custom CNN, transfer learning, evaluation, YOLOv8 & Streamlit notes.
- `train_classification.py` — runnable script to train the classification models (custom or transfer).
- `requirements.txt` — recommended Python packages.
- `object_detection_Dataset/data.yaml` — YOLOv8 data config (relative paths) for training detection models.
- `app.py` (generated by the notebook) — minimal Streamlit app to load a saved model and predict uploaded images.

Quick setup (Windows PowerShell):

1) Create a Python virtual environment and activate it:

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
```

2) Upgrade pip and install dependencies (CPU TF recommended unless you have GPU setup):

```powershell
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
# If you have no GPU and prefer the CPU-only TF wheel, install:
# python -m pip install tensorflow-cpu
```

3) Run the notebook (`Aerial Object Classification.ipynb`) in Jupyter or VS Code. Cells that require TensorFlow are guarded; once TF is installed re-run from the top.

4) To train from the script (example):

```powershell
python train_classification.py --model custom --data_dir classification_dataset --epochs 10 --batch_size 32
```

5) To run the Streamlit demo (after saving a model named `best_custom_cnn.h5` in the workspace):

```powershell
streamlit run app.py
```

6) YOLOv8 (optional):
- Install `ultralytics` (recommended in the same venv): `python -m pip install ultralytics`.
- Ensure `object_detection_Dataset/data.yaml` paths are correct relative to where you run the command.
- Train with a small model for quick tests (example):

```powershell
# run from project root
yolo detect train data=object_detection_Dataset/data.yaml model=yolov8n.pt epochs=50 imgsz=640
```

Notes & troubleshooting:
- The environment used by this editor may not allow pip installs; if package installs fail, run the install commands locally on your machine.
- The notebook cells include guards that skip TF-dependent code if TensorFlow is not present — this allows inspection and visualization to run even when heavy packages are missing.
